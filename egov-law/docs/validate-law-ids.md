# law_id検証手順

このドキュメントでは、law-aliases.mdに記載された全法令のlaw_idをe-Gov API V2で検証する手順を説明します。

## 概要

law-aliases.mdに記載されている38件の法令IDが、実際のe-Gov API V2で有効であることを確認します。

## 検証方法

### 方法1: 自動検証スクリプト（推奨）

`scripts/validate-law-ids.sh`を使用して、全law_idを一括で検証できます。

```bash
cd egov-law
bash scripts/validate-law-ids.sh
```

**実行条件:**
- インターネット接続が必要
- `curl`コマンドが利用可能であること
- e-Gov API V2 (`https://laws.e-gov.go.jp/api/2/`) にアクセス可能であること

**出力例:**
```
===========================================
law-aliases.md 法令ID検証スクリプト
===========================================

検証対象: 38 件の法令ID

[OK] 321CONSTITUTION - 日本国憲法
[OK] 129AC0000000089 - 民法
[OK] 140AC0000000045 - 刑法
...
===========================================
検証結果サマリー
===========================================
総件数: 38
成功: 38
失敗: 0

✓ すべての法令IDが有効です
```

### 方法2: 個別検証

既存のスクリプトを使って個別に検証することもできます。

```bash
# 改正履歴取得で法令IDの有効性を確認
bash scripts/fetch-revisions.sh 129AC0000000089
```

正常な場合、法令の改正履歴がJSON形式で返されます。
無効な法令IDの場合、エラーメッセージが返されます。

## 検証内容

各law_idに対して以下を確認:

1. **APIレスポンスの成功**: HTTP 200が返ること
2. **法令名の取得**: `law_title`フィールドが存在すること
3. **期待通りの法令名**: 正式名称と一致すること

## トラブルシューティング

### エラー: "Could not resolve host"

DNSが解決できない場合、以下を確認:
- インターネット接続が有効か
- プロキシ設定が必要か
- ファイアウォールでブロックされていないか

### エラー: HTTP 400 - "法令ID、法令番号、又は法令履歴IDが誤っています"

law_idが無効な場合のエラーです。以下を確認:
- law-aliases.mdの記載が正しいか
- APIが最新のlaw_id形式に対応しているか
- 法令が廃止されていないか（廃止済み法令でもAPIは応答する場合があります）

### エラー: HTTP 500

e-Gov API側の一時的なエラーの可能性があります。
時間をおいて再実行してください。

## 注意事項

### API呼び出し頻度

validate-law-ids.shは、各APIリクエストの間に0.5秒の間隔を設けています。
これは、e-Gov APIサーバーへの負荷を軽減するためです。

大量のリクエストを短時間に送信すると、一時的にアクセスが制限される可能性があります。

### 廃止済み法令

law-aliases.mdには廃止済み法令（「IT基本法」など）も含まれています。
廃止済みでもe-Gov APIで参照可能な場合があります。

## 検証完了後の対応

### すべてのlaw_idが有効な場合

- issue #12 を完了としてクローズ
- 検証結果をissueにコメント

### 無効なlaw_idが見つかった場合

1. e-Gov API V2で正しいlaw_idを検索
   ```bash
   bash scripts/search-laws.sh "法令名"
   ```

2. law-aliases.mdを修正してPR作成

3. 再検証を実施
